# Data

This directory contains configuration data used by the game.

Instead of hardcoding gameplay values in scripts, data files define parameters such as:

Examples:
- weapons
- mech parts
- enemy templates
- area definitions
- encounters
- rewards
- factions

Gameplay values should live here whenever possible rather than being hardcoded in scripts.
This should make balancing, iteration, and future expansion easier.

Separating gameplay data from logic makes balancing and iteration easier
